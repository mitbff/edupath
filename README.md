# EduPath 智能导学 Agent

这是一个面向“智慧教育”期末大作业的可演示原型。系统围绕“在线答题 → 错因诊断 → 知识追踪 → 知识图谱 → 个性化推荐 → 教师干预”构建完整闭环。

新版已经从“单一导数场景”升级为“多课程场景配置平台”：同一套诊断引擎可以切换到数学导数、英语阅读、Python 编程等不同场景。真实投入应用时，只需要由教师维护题库、知识点图谱和资源库，就可以扩展到更多课程。

## 推荐选题名称

**EduPath 智能导学 Agent：面向多课程场景的知识图谱驱动学情诊断系统**

更正式的论文式题目：

**面向智慧教育的知识图谱驱动学情诊断与个性化学习路径推荐系统设计与实现**

## 如何运行

直接双击打开：

```text
index.html
```

系统使用 ECharts CDN。如果图表没有显示，说明当前网络无法访问 CDN，可以下载 ECharts 到本地后替换 `index.html` 中的脚本地址。

## 已实现功能

- 多场景配置模块：支持数学导数、英语阅读、Python 编程三个演示场景。
- 在线答题模块：每个场景提供一组诊断题，记录学生答案、答题用时和正确率。
- 错因诊断模块：每道题绑定知识点、难度、正确答案和典型错因标签。
- 知识图谱模块：用 ECharts graph 展示不同课程的知识依赖关系。
- 知识追踪算法：提交后根据答题表现动态更新每个知识点掌握度。
- 个性化推荐模块：根据最薄弱知识点推荐微课、例题、巩固题和错题复盘任务。
- AI 讲解 / 练习生成模块：用规则模板模拟大模型讲解，自动生成下一组针对性练习。
- 教师端看板：包含班级知识点热力图、高频错因排行、学生分层、教学建议和分层作业。
- 高级诊断 Agent：生成诊断证据链、自适应下一题策略和资源匹配理由。
- 学习风险预警：结合掌握度、错因密度和学生分层，生成干预优先级。

## 已支持的应用场景

```text
1. 高一数学 · 导数单元
   知识链：函数基础 → 极限思想 → 导数定义 → 单调性 → 极值应用 → 导数应用

2. 高中英语 · 阅读理解
   知识链：词汇识别 → 语境推断 → 长难句分析 → 信息定位 → 主旨概括

3. Python 编程 · 入门项目
   知识链：变量与类型 → 条件分支 → 循环结构 → 函数封装 → 项目应用
```

这三个场景共用同一套算法和页面，只替换配置数据。这一点可以用来回应“是否只能用于一个章节”的问题。

## 核心算法

### 1. 知识追踪更新

```text
新掌握度 = 旧掌握度 × 0.7 + 本次表现 × 0.3
```

### 2. 本次表现分

```text
表现分 = 正确率 × 0.5 + 难度系数 × 0.2 + 稳定性 × 0.3
```

本原型中：

- 正确率来自学生提交答案。
- 难度系数来自题目元数据。
- 稳定性由答题总耗时与预期耗时差距估算。

### 3. 错因诊断标签

- 概念不清
- 公式误用
- 审题偏差
- 计算失误
- 方法选择不当
- 未作答

## 汇报演示流程

1. 打开系统总览，介绍它不是普通考试系统，而是诊断和干预闭环。
2. 进入“智能答题”，点击“填入演示答案”，再提交诊断。
3. 展示学生报告：正确率、用时、错因解释、知识雷达、推荐资源、AI 练习生成。
4. 回到总览，展示知识图谱节点颜色变化。
5. 进入教师看板，展示热力图、错因排行、学生分层和自动生成分层作业。

## 真实应用落地方式

当前版本是静态原型，适合期末汇报演示；真实投入应用时建议升级为“前端 + 后端 + 数据库 + AI 服务”的平台架构。

### 需要补齐的生产能力

- 用户系统：学生、教师、管理员登录与权限控制。
- 题库管理：教师上传题目、设置答案、知识点、难度和错因标签。
- 场景配置：管理员维护课程、单元、知识图谱和资源库。
- 学习记录：保存学生每次作答、掌握度变化和推荐结果。
- AI 服务：接入真实大模型 API，生成更自然的讲解、练习和教师建议。
- 数据安全：脱敏学生信息，控制数据访问权限。
- 部署运维：后端服务、数据库备份、日志监控和异常处理。

### 真实系统数据流

```text
教师配置课程场景
→ 导入题库和知识图谱
→ 学生在线答题
→ 后端记录作答日志
→ 诊断服务更新掌握度
→ 推荐服务生成学习路径
→ AI 服务生成讲解和练习
→ 教师端查看班级学情和分层作业
```

## 技术栈建议

当前原型：

- HTML + CSS + JavaScript
- Apache ECharts
- 多场景配置数据
- 规则引擎模拟 AI 诊断和练习生成
- 可选 FastAPI + SQLite 后端，用于接口、作答日志和掌握度快照持久化

可升级版本：

- 前端：Vue 3 / React + ECharts
- 后端：FastAPI，本项目已提供轻量版后端目录 `backend`
- 数据库：SQLite / MySQL，本项目后端已使用 SQLite 自动建表
- 算法：知识追踪、错因诊断、学生分层聚类
- AI：接入大模型 API 生成讲解和练习

## 创新点

- 从“只给分数”升级为“解释错因 + 推荐路径 + 教师干预”。
- 从“单一课程 demo”升级为“多场景可配置平台”。
- 用知识图谱表达不同课程的知识依赖关系，薄弱节点可视化更直观。
- 用简化 BKT 思路实现动态掌握度更新，答辩时有明确算法公式。
- 用证据链解释诊断来源，避免 AI 建议像黑箱。
- 用自适应出题策略控制下一题难度，让推荐不只是资源列表。
- 同时覆盖学生端和教师端，体现智慧教育中的个性化学习与精准教学。
- AI 模块先用规则模板模拟，后续可无缝接入真实大模型。
